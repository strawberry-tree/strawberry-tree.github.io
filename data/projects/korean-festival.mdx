## 프로젝트 개요

전국의 다양한 축제 정보를 통합하여 제공하는 웹 플랫폼입니다. 사용자는 날짜별 달력 뷰를 통해 축제 일정을 직관적으로 확인하고, 상세 페이지에서 축제에 대한 모든 정보를 얻을 수 있습니다. React와 Vite를 기반으로 빠른 성능과 부드러운 사용자 경험을 제공하며, MongoDB를 활용한 효율적인 데이터 관리가 특징입니다.

## 기술적 성과

### 1. 메인 페이지 - 캐러셀 기반 시각적 UX 구현

**문제**: 사용자가 메인 페이지에 진입했을 때 축제 정보를 시각적으로 효과적으로 전달하면서도 가독성을 유지해야 했습니다.

**해결**: 흐릿한 배경 이미지 위에 축제 포스터를 겹쳐 배치하고, 반투명 카드를 통해 축제 정보를 표시하는 듀얼 레이아웃 캐러셀을 구현했습니다.

- **레이어 구조**: 배경 이미지(`blur(2px) brightness(0.7)`), 포스터 이미지(absolute positioning), 정보 카드(`rgba(0,0,0,0.3)`)의 3단계 레이어
- **포스터 위치**: `left: 28%`, `transform: translate(-50%, -50%)`로 중앙에서 왼쪽으로 이동
- **정보 카드**: `left: 45%`, `min-width: 400px`로 포스터 오른쪽에 배치, `box-shadow`로 부유감 표현
- **반응형 디자인**: 데스크톱에서는 듀얼 레이아웃, 모바일에서는 중앙 정렬 단일 카드로 자동 전환
- **오류 처리**: 이미지 로드 실패 시 placeholder 이미지로 대체

**결과**: 시각적으로 풍부한 UI와 함께 축제의 핵심 정보를 직관적으로 제공하여 사용자 참여도를 높였습니다.

### 2. 달력 페이지 - 인터랙티브 월별 축제 달력

**문제**: 사용자가 날짜를 선택했을 때 해당 날짜의 축제 목록을 빠르게 확인할 수 있는 직관적인 탐색 경험이 필요했습니다.

**해결**: 달력 UI에서 날짜 클릭 시 해당 날짜의 축제 목록으로 자동 스크롤되는 인터랙티브 기능을 구현했습니다.

- **상태 관리**: `currentDate` state로 선택된 날짜 추적, `shouldScroll` ref로 스크롤 트리거 관리
- **달력 생성 로직**: 해당 월의 첫 날짜를 기준으로 일요일부터 시작하는 6주(42일) 그리드 생성
- **월별 축제 데이터**: `getMonthFestivals` API를 통해 월 단위로 일별 축제 개수를 가져와 Badge로 표시
- **날짜 선택 효과**:
  - 선택된 날짜: `background-color: #e3f2fd`, `border: 2px solid #2196f3`
  - 오늘 날짜: `background-color: #fff3cd`, `border: 1px solid #ffc107`
  - 호버 효과: `background-color: #f8f9fa`
- **자동 스크롤**: 날짜 클릭 시 `shouldScroll.current = true` 설정, `useEffect`에서 축제 목록 로드 완료 후 `scrollTo(festivalListRef)` 실행

**결과**: 사용자가 달력에서 날짜를 선택하면 즉시 해당 날짜의 축제 목록으로 부드럽게 이동하여 탐색 효율성이 향상되었습니다.

### 3. 축제 상세 페이지 - 몰입형 전체화면 배경 효과

**문제**: 축제 상세 정보를 제공하면서도 축제의 분위기를 시각적으로 전달할 수 있는 차별화된 UI가 필요했습니다.

**해결**: 축제의 썸네일 이미지를 전체 배경으로 고정(`background-attachment: fixed`)하여 스크롤 시에도 유지되는 몰입형 UI를 구현했습니다.

- **배경 이미지 적용**: `useEffect`로 컴포넌트 마운트 시 `document.body.style.backgroundImage`에 썸네일 URL 설정
- **고정 배경 효과**: `background-attachment: fixed`, `background-size: cover`, `background-position: center`로 스크롤과 무관하게 배경 고정
- **콘텐츠 레이아웃**:
  - 상단 95vh 공백으로 초기 진입 시 배경 이미지만 표시
  - 콘텐츠 영역은 흰색 배경으로 구분하여 가독성 확보
- **헤더 스타일 동적 변경**:
  - 배경 이미지 위에서 헤더를 투명하게 처리
  - 네비게이션 링크와 브랜드를 흰색으로 변경
  - 로고 이미지 반전(`filter: invert(1) brightness(1)`)으로 가시성 확보
- **클린업 로직**: 컴포넌트 언마운트 시 모든 body와 헤더 스타일 원복

**결과**: 축제 상세 페이지 진입 시 즉시 축제의 분위기를 느낄 수 있는 풀스크린 배경과 함께, 스크롤 시에도 배경이 고정되어 극적인 효과를 제공합니다.
