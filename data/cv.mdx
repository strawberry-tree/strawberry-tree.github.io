<Tag text="Next.js" />
<Tag text="NestJS" /> <Tag text="React Three Fiber" /> <Tag text="OpenCV.js" />
<Tag text="Socket.IO" /> <Tag text="Redis" /> <Tag text="PostgreSQL" />

- **팀 리더**로서 기획, 개발, 배포 전 과정 총괄
- [**GLB 파일 로딩 최적화**](https://strawberry-tree.github.io/projects/wheretoput#glb-loading)
  - **문제**: AI 생성 대용량 3D 모델(.glb)로 인한 시뮬레이터 로딩 지연
  - **해결**: 텍스처 포맷 WebP 변환 후 Redis 캐싱
  - **파일 압축률 92%**(평균 1.3MB → 0.1MB), **로딩 속도 58% 개선**(6s → 2.5s)
- [**도면 자동 인식**](https://strawberry-tree.github.io/projects/wheretoput#opencv-floor-plan)
  - **문제**: 사용자가 직접 도면의 벽을 그리는데 약 1분 소요
  - **해결**: OpenCV.js Hough Transform 기반 선 검출 + 평행선 병합 → **3초 내 벽 자동 검출**
- [**벽 자석 기능**](https://strawberry-tree.github.io/projects/wheretoput#wall-magnet)
  - **문제**: 3D 환경 내 가구를 벽에 맞춰 정렬하는 작업의 번거로움
  - **해결**: OBB 기반 가구-벽 거리 계산 → 근접 시 가구가 벽에 달라붙어 **배치 편의성 향상**
  - 코너 벽 자석 기능 구현 과정에서 성능 최적화 (**O(N²) -> O(N)**)
- [**실시간 협업 안정화**](https://strawberry-tree.github.io/projects/wheretoput#collaboration-lock)
  - **문제**: 다중 사용자가 동일 가구 동시 수정 시 충돌
  - **해결**: Socket.IO + Redis Hash 기반 락 시스템 구현 → **동시 수정 방지**
