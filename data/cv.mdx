## 자기소개
### 성능 개선에 집중하는 프론트엔드 개발자

저는 좋은 사용자 경험을 주기 위해서 성능 최적화가 필수적이라고 믿습니다. 특히 프론트엔드는 사용자가 직접 체감하는 영역이기에, 기술적 고민을 통한 성능 개선이 즉각적인 만족도로 이어진다는 점이 매력적입니다. 크래프톤 정글 최종 프로젝트로, React Three Fiber 기반 [3D 인테리어 시뮬레이터 '어따놀래'](https://strawberry-tree.github.io/projects/wheretoput)를 개발하며 성능 개선을 고민했던 경험이 있습니다. 버즈빌의 게임 광고 초대 및 참여 상품은 성능 개선이 곧 사용자의 체감 만족도와 참여율 증가로 이어지는 환경입니다. 이는 성능 최적화에 집중하는 저에게 최적의 환경이라고 생각합니다.

### 숨겨진 병목을 발견하고 성능을 극대화하다

'어따놀래'를 개발하며, 가구를 벽 교차점 근처에 자동으로 달라붙게 하는 코너 스냅 기능을 구현했습니다. 하지만 초기 설계 는 모든 두 벽의 조합을 계산하는 O(N²)을 필요로 하여, 벽 개수가 늘어날수록 잠재적 병목이 심화될 수 있었습니다. 따라서 벽을 수직 / 수평 그룹으로 나눠 각각 한 번만 순회하고, 각 그룹에서 가구와 가장 가까운 벽을 선택하는 O(N) 로직으로 재설계하였습니다. 이러한 알고리즘 효율성 확보를 통해, 벽의 개수가 많아질 때 생길 수 있는 잠재적 병목을 미리 해결할 수 있었습니다.

### 프론트엔드, 백엔드 간 협업 시너지

'어따놀래' 프로젝트의 리더로, 다양한 강점을 가진 동료 간 소통을 조율했습니다. 3D 모델에 사용된 GLB 파일 로딩 최적화 과정에서 제가 텍스처 포맷을 PNG에서 WebP로 변환하여 용량을 92% 압축하자, 백엔드 동료가 Redis 캐싱도 현실적이겠다며 추가 최적화를 제안했습니다. 결국 프론트엔드 개선이 백엔드 전략으로 확장되며, 시뮬레이터 로딩 시간을 6초에서 2.5초로 58% 개선하는 시너지를 경험했습니다. 각자의 영역에서 문제를 발견하고 해결책을 제시할 때 최고의 결과가 나온다는 것을 체감하며, 기술적 협업의 힘을 깊이 깨달았습니다.

## 자기소개
### 학습의 심리학을 기술로 구현하기
시험공부를 하면서 분명 이해했던 내용이 시험장에서는 떠오르지 않는 경험을 자주 했습니다. 이때 학습심리학 전공 수업에서 배운 "testing effect", 즉 퀴즈를 통해 기억을 점검하면 장기기억이 강화된다는 원리가 떠올랐습니다. "LLM을 통해 퀴즈를 자동 생성하면, 학습에 효과적이지 않을까?"라는 질문에서 시작해 [직접 모의고사 서비스를 만들어 보았습니다](https://strawberry-tree.github.io/projects/test-maker).

### 사용자 반응에서 발견한 LLM의 잠재력

GPT-4 API와 Langchain을 연동해, 학습 자료를 업로드하면 자동으로 객관식 문제와 해설을 생성하도록 프롬프트를 설계하여, Streamlit Cloud에 배포했습니다. 친구들에게 공유하자, <strong>"장소나 시간에 구애받지 않아 공부에 효율적이다."</strong>, <strong>"직접 만든 정리본에서 문제를 생성해 줘서 자율 학습에 큰 도움이 된다."</strong>는 긍정적 반응을 받으며, LLM이 인간의 학습능력을 향상할 수 있음을 확인했습니다. 특히 <strong>"저는 소수 과목 임용고시를 준비 중인데, 참고서가 부족한 분야라 서비스가 매우 유용했다"</strong>는 피드백에서, AI가 소수가 겪는 불편함도 해결할 수 있다는 가능성을 발견했습니다.

### 사용자 관점에서 생각하는 Service Architect

<strong>"사실 확인 수준의 문제라 암기 과목에만 효과적이다"</strong>,
<strong>"정답과 오답 선지가 지나치게 구분된다"</strong>는 피드백도 있었습니다. 이를 통해 AI 서비스를
개발할 때는, 기술적인 문제해결력뿐만 아니라, 실제 사용자의 행동과 불편을 빠르게 파악하고 개선하는 것
역시 중요함을 배웠습니다. 이번 경험을 바탕으로, 노타 AI에서 Service Architect로서 문제를 발견하고
AI를 활용해 해결책을 만들며, 사용자 피드백을 반영해 개선 방향을 찾아가는 선순환을 실현하고 싶습니다.

### 도면 벽 검출 구현

크래프톤 정글 최종 프로젝트로 [3D 인테리어 시뮬레이터 '어따놀래'](https://strawberry-tree.github.io/projects/wheretoput)를 개발하며, 2D 도면을 업로드하면 벽을 인식해 검출하는 기능을 구현해야 했습니다. 범용적인 OpenCV.js 라이브러리를 활용할지, CNN 기반 기계학습 모델에 도면 이미지 데이터를 파인튜닝하여 특화된 모델을 구축할지 선택해야 하는 상황에서, 데이터 기반의 비교분석을 진행했습니다.

### 시간, 비용, 사용자 경험 간 균형을 고려한 의사결정

OpenCV.js는 범용 라이브러리인 만큼 바로 사용할 수 있지만, 도면 분석에 특화되지 않아 일부 오차가 발생할 수 있었습니다. 반면 DeepLabV3+ CNN 모델을 도면 데이터에 파인튜닝하면 인식 정확도는 높아지지만, 데이터 준비와 GPU 학습에 최소 1주가 소요되고, GPU 서버 운영에도 월 400달러의 비용이 발생할 것으로 예상됐습니다. 짧은 프로젝트 기간을 고려했을 때, OpenCV.js로 즉시 처리한 뒤 사용자가 직접 수정하는 방식이 더 효율적이라 판단했습니다. 따라서 그림판 기반 벽 그리기와 삭제 같은 편집 기능을 추가해 사용자가 오차를 쉽게 보정할 수 있도록 했습니다. 기술의 한계를 UX 디자인으로 보완한 접근이었습니다.

### 현실적 제약 속 최선의 의사결정

이 경험을 통해 완벽한 기술적 구현보다, 데이터 기반 분석을 통해 현실적인 제약 속에서 최선의 선택을 하는 것이 중요함을 느꼈습니다. 특히 기술적 완성도와 사용자 경험, 그리고 프로젝트 일정이라는 트레이드오프 속에서 우선순위를 명확히 하는 법을 배웠습니다. 아바티에서도 데이터 기반 분석을 통해 효율적인 답안을 찾는 개발자가 되겠습니다.

</section>

### 가장 어려웠던 문제 해결 경험 - GLB 파일 로딩 성능 최적화
[3D 인테리어 시뮬레이터 '어따놀래'](https://strawberry-tree.github.io/projects/wheretoput) 개발 중, 대용량 3D 모델 파일 로딩 지연 문제를 마주했습니다. 우선 DRACO 압축을 시도했지만, 압축률은 약 10%로 미미했습니다. 이후 GLB 파일의 텍스처가 고용량의 PNG 포맷임을 확인하고, WebP로 변환하여 평균 파일 크기를 0.1MB로 92% 감소시켰습니다. 이후 동료의 제안으로 압축된 파일을 Redis에 캐싱한 결과, 로딩 시간을 6초에서 2.5초로 58% 개선했습니다. 리소스 최적화가 백엔드 캐싱 전략으로 확장되며 협업의 시너지를 체감했습니다.

### 사용자 입장에서 고민한 기능 개선 경험 - 벽 자석 기능 개발

앞선 ['어따놀래'](https://strawberry-tree.github.io/projects/wheretoput) 프로젝트에서 가구를 벽에 정확히 배치하는 작업이 번거로웠습니다. 3D 환경에선 정밀한 마우스 조작이 필요했고, 오차 발생 시 가구가 벽을 뚫고 나가기도 했기 때문입니다. 파워포인트나 피그마의 길이 계산을 통한 자동 정렬 기능을 떠올려, 벽에 가구가 가까이 닿을 때 자동으로 달라붙는 벽 자석 기능을 구현했습니다. 결국 별도 라이브러리 없이, 편리한 가구 배치를 가능하게 하여 사용성을 개선했습니다.

### 최고가 되기 위한 노력 - 교육봉사에서 배운 진정한 성취의 의미

[5박6일 방문 교육봉사 '나눔교실'](https://drive.google.com/file/d/1UKpWGjgqVjfx1iCKnvOxczPR0S2P0c3F/view?usp=sharing)에서 '학생들에게 최고의 경험을 줄 수 있는 방법'을 고민했습니다. 이때 단순한 지식 전달보다, 학생들과 마음을 열고 소통하는 것을 중점에 두었습니다. 수업 준비는 물론, 상담을 통한 진솔한 대화를 통해 가르침의 본질이 일방적 전달이 아닌 쌍방향 소통임을 배웠습니다. 마지막 날엔 학생들을 위한 특별 합창 무대를 이끌며, 처음 해보는 역할에 두려움도 있었지만, 동료 선생님들과 협력하며 성공적으로 끝맺었습니다. 최고가 된다는 것은 개인 성과가 아니라 함께 의미 있는 가치를 만드는 것임을 깨달았고, 이는 이후 팀 프로젝트에서 동료 성장을 함께 고민하는 태도로 이어졌습니다.
